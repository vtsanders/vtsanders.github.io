How I made Da Venchei Codes

Creating a beautiful personal website with Blogdown, HUGO, and RMarkdown

Aug 17, 2020 3 min read

So you are thinking about creating a website? Good. Blogdown, HUGO, and RMarkdown are here to save us!

I’m assuming you have working knowledge of R, Markdown, and can leverage Stack Overflow and related tech message boards for help. Moreover, these instructions are not all inclusive and may leave out steps - hence, one’s ability to leverage tech message boards would be useful. This guide should (I) get you up and running locally, then (II) through deployment via Github Pages or Google Domains.

To start, we’re going to need a theme from Hugo. I chose the Academic theme for Da Venchei Codes. Next, lets open RStudio and start an R Project. An R Project is useful because RStudio will recognize your project as a website and surface the appropriate build tools in the interface.

For this walk-through, I’m going to use the Navigator Hugo Theme. Let’s start by creating a new folder in documents or on the desktop. I’m naming it “davenchei_examplesite”.

Next, let’s open RStudio then, create a new project then, create a directory, and download the theme and it’s dependencies.

Now select New Directory then scroll to Website using blogdown.

We will create the website as part of a subdirectory to /davenchei_examplesite/. Enter a name for the directory -I chose “source_files”; then enter the theme’s github address -“themefisher/navigator-hugo”. You can also clone the repo, or follow the instructions here. You can load the demo site into the Viewer by clicking “Addins” then “Serve Site” buttons.

In the viewer, you’ll see the mobile version of the demo site. Let’s quickly modify the underlying files by going to the pane containing “Files, Plots, Packages, Help, etc”. Navigate to the directory containing the project and click on the “config.toml” file. This configuration file contains the tools needed to manipulate the basic structure for this theme. Here you would insert Google Analytics for User Engagement event tracking, or change the website URL (at the top), Page Title, etc. I highly suggest purchasing a website domain with Google Domains and/or purchasing a GSuite account to own both email and website domain for your direct communication to external parties.

Since we’ll be hosting via github pages, let’s start by changing the Publishing Directory to “../davenchei_examplesite.github.io”. This will be the folder that we will push to github for deployment. Moreover, changing the publishing directory allows for greater flexibility in terms of having control over the security and version control for the site - all data and changes are local until deployment. You can created a private Repo and change that in github later if you’d like.

Let’s change the title by going to the section labeled “# Hero Area Section” and replacing the heading from “We love to make brands grow” to “We love Da Venchei Codes”. Let’s change the button text from “Know More” to “Meet Da Venchei”. Finally, Let’s change the URL from https://themefisher.com/ to https://www.davencheicodes.com/ and the description from “Lorem ipsum dolor…” to “This is a Da Venchei Tutorial.”

URL from “https://example.com/” to “https://davenchei_examplesite.github.io/”.
